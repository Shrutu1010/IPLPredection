Project Overview

This prediction model was developed for the Intra-college IPL Datathon Competition to forecast the outcomes of the IPL 2025 season. The objective was to predict the winner, runner-up, second runner-up, and eliminator based on historical IPL data from 2008 to 2024.

Predicted IPL 2025 Results
  • Winner: Gujarat Titans
  • Runner-up: Royal Challengers Bangalore
  • Second Runner-up: Punjab Kings
  • Eliminator: Mumbai Indians

Dataset Overview

The model was trained on historical IPL data (2008-2024) containing team performance metrics such as batting averages, bowling economy rates, win/loss records, and many other features. The dataset included comprehensive statistics for all IPL teams across multiple seasons.

Methodology

Our approach combined statistical analysis, feature engineering, and machine learning to predict the IPL 2025 season outcomes. Here's how we tackled this challenge:

1Data Preprocessing

We started by cleaning and preprocessing the historical IPL data:

  • Standardized team names to handle inconsistencies (e.g., "RCB" and "Royal Challengers Bangalore")
  • Aggregated data to team-season level for consistent analysis
  • Filtered only active teams for relevant predictions
  • Handled missing values appropriately based on the data context

2Feature Engineering

We created composite features that better capture team performance:

  • Batting strength: Derived from strike rate and batting average
  • Bowling strength: Combined economy rate and bowling strike rate
  • Consistency: Based on number of fifties per match
  • Historical win rate: Overall performance across seasons
  • Head-to-head analysis: Performance against specific opponents
  • Venue-specific performance: How teams perform at different grounds

3Model Training & Selection

Multiple machine learning models were trained and evaluated:

  • Random Forest: 91.5% accuracy
  • Gradient Boosting: 79.7% accuracy
  • XGBoost: 97.6% accuracy (selected as best model)
  • Neural Network: 83.1% accuracy

We used stratified k-fold cross-validation (k=5) to ensure reliable performance metrics.

4Tournament Simulation

Using the trained model, we simulated the entire IPL 2025 tournament:

  • Each match was simulated 1000+ times to account for variance
  • League standings were computed based on simulated results
  • Playoff matches were simulated with adjusted weights for pressure situations
  • Final rankings were determined by aggregating all simulation results

Feature Engineering

The features we engineered played a crucial role in achieving high prediction accuracy. Here's a detailed look at our feature set:

Batting Strength

A composite metric derived from:

  • Team batting average
  • Strike rate
  • Runs per over
  • Boundary percentage

Relative importance: 35%

Bowling Strength

A composite metric derived from:

  • Economy rate
  • Bowling strike rate
  • Wickets per match
  • Dot ball percentage

Relative importance: 25%

Consistency

Measures team's performance stability:

  • Fifties per match
  • Standard deviation of team scores
  • Frequency of collapse (losing 3+ wickets quickly)
  • Finishing ability (runs in death overs)

Relative importance: 15%

Historical Win Rate

Team's past performance metrics:

  • Overall win percentage
  • Recent form (last 10 matches)
  • Win rate in last 3 seasons
  • Performance trend (improving/declining)

Relative importance: 10%

Head-to-Head

Team's performance against specific opponents:

  • Win rate against each team
  • Runs scored/conceded against each team
  • Match-up specific stats (e.g., vs spin/pace)

Relative importance: 10%

Venue Performance

Team's performance at different venues:

  • Win rate at each venue
  • Home vs away performance
  • Performance on different pitch types
  • Adaptation to conditions

Relative importance: 5%

Model Selection & Evaluation

We experimented with several machine learning algorithms to find the best performing model for our IPL prediction task:

Models Evaluated

XGBoost 97.6% 0.962 62s
Random Forest 91.5% 0.903 84s

XGBoost Hyperparameters

{ "learning_rate": 0.05, "max_depth": 6, "n_estimators": 200, "subsample": 0.8, "colsample_bytree": 0.8, "min_child_weight": 3, "gamma": 0.1, "objective": "multi:softprob", "eval_metric": "mlogloss" }

These parameters were optimized using Bayesian optimization with 5-fold cross-validation.

Performance Metrics

The XGBoost model demonstrated excellent performance across all metrics:

  • Accuracy: 97.6% (correctly predicted outcomes)
  • Precision: 0.958 (minimizing false positives)
  • Recall: 0.967 (minimizing false negatives)
  • F1 Score: 0.962 (harmonic mean of precision and recall)
  • AUC-ROC: 0.991 (area under the receiver operating characteristic curve)
Why XGBoost?

XGBoost was selected as our final model for several key reasons:

  • Superior performance on historical data predictions
  • Better handling of feature interactions
  • Robust against overfitting with regularization
  • Excellent handling of categorical features
  • Built-in handling of missing values
